1. `packages/collectors/src/utils/naver-parser.ts` -- URL 파싱 유틸:

```typescript
// 네이버 뉴스 기사 URL에서 oid(언론사 코드)와 aid(기사 번호) 추출
// URL 패턴: https://n.news.naver.com/article/{oid}/{aid}
// RESEARCH Pitfall 6 참고: objectId 형식이 news{oid},{aid}
export function parseNaverArticleUrl(url: string): { oid: string; aid: string } | null {
  // 패턴 1: https://n.news.naver.com/article/{oid}/{aid}
  const match1 = url.match(/n\.news\.naver\.com\/article\/(\d+)\/(\d+)/);
  if (match1) return { oid: match1[1], aid: match1[2] };

  // 패턴 2: https://news.naver.com/main/read.nhn?oid=xxx&aid=xxx
  const match2 = url.match(/[?&]oid=(\d+).*[?&]aid=(\d+)/);
  if (match2) return { oid: match2[1], aid: match2[2] };

  return null;
}

// 네이버 댓글 API용 objectId 생성
export function buildObjectId(oid: string, aid: string): string {
  return `news${oid},${aid}`;
}

// 네이버 뉴스 검색 URL 생성
// 검색 파라미터: query(키워드), ds/de(시작/종료 날짜 YYYY.MM.DD), start(페이지 오프셋), sort(0=관련도, 1=최신, 2=오래된순)
export function buildNaverSearchUrl(params: {
  keyword: string;
  startDate: string; // ISO 8601
  endDate: string; // ISO 8601
  page: number; // 1-based
  sort?: number; // 0=관련도, 1=최신
}): string {
  const ds = new Date(params.startDate)
    .toLocaleDateString('ko-KR', { year: 'numeric', month: '2-digit', day: '2-digit' })
    .replace(/\. /g, '.')
    .replace('.', '');
  const de = new Date(params.endDate)
    .toLocaleDateString('ko-KR', { year: 'numeric', month: '2-digit', day: '2-digit' })
    .replace(/\. /g, '.')
    .replace('.', '');
  const start = (params.page - 1) * 10 + 1;
  const sort = params.sort ?? 1; // 기본 최신순
  return `https://search.naver.com/search.naver?where=news&query=${encodeURIComponent(params.keyword)}&sm=tab_opt&sort=${sort}&ds=${ds}&de=${de}&start=${start}`;
}
```

비공식 네이버 댓글 API(`apis.naver.com/commentBox/cbox/web_naver_list_jsonp.json`)를 사용하여 기사별 댓글을 수집한다.

구현 핵심:

- `implements Collector<NaverComment>` -- Collector 인터페이스 구현
- `source = 'naver-comments'`
- `collect()` AsyncGenerator -- 기사 단위로 댓글을 yield
- collect 옵션에 기사 URL 목록을 전달받거나, 별도 메서드 `collectForArticle(url: string, options)` 제공

API 호출 구현 (RESEARCH.md Code Examples 참고):

- URL: `https://apis.naver.com/commentBox/cbox/web_naver_list_jsonp.json`
- 필수 파라미터: ticket=news, pool=cbox5, lang=ko, objectId=`news{oid},{aid}`, pageSize=100, page={n}, sort=FAVORITE|NEW|OLD
- 필수 헤더 (RESEARCH Pitfall 1):
  - `Referer: https://n.news.naver.com/article/comment/{oid}/{aid}`
  - `User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36`
- 응답은 JSONP 형식 가능 -- `_callback({...})` 래퍼 제거 후 JSON.parse
- 페이지네이션: page 1부터 시작, `result.pageModel.totalPages`로 전체 페이지 확인
- maxComments(D-03 기본 500개) 도달 시 중단
- 요청 간 1-3초 딜레이 (RESEARCH Pitfall 2)
- `parseNaverArticleUrl()`로 URL에서 oid/aid 추출
