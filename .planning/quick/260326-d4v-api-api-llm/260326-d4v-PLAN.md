# Quick Plan: API 키 관리 ↔ 모듈별 모델 설정 연동

## Task 1: tRPC settings.update provider enum 확장

**파일:** `apps/web/src/server/trpc/routers/settings.ts`

현재 `z.enum(['anthropic', 'openai'])`만 허용 → providerKeys에서 지원하는 모든 타입으로 확장.

## Task 2: model-settings.tsx에서 providerKeys 데이터 연동

**파일:** `apps/web/src/components/settings/model-settings.tsx`

- 하드코딩된 `PROVIDER_MODELS` 제거
- `providerKeys.list` 쿼리 추가하여 등록된 프로바이더와 selectedModel 목록을 동적으로 구성
- 프로바이더 드롭다운: 등록된 키의 providerType 목록
- 모델 드롭다운: 해당 providerType의 등록된 키들의 selectedModel 목록
- 등록된 키가 없으면 안내 메시지 표시

## Task 3: provider 변경 시 첫 번째 모델 자동 선택 로직 수정

기존 `PROVIDER_MODELS[provider][0]` 대신 동적 모델 목록의 첫 번째 항목 사용.
