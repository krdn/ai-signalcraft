1. `packages/collectors/src/utils/youtube-client.ts` -- googleapis YouTube 클라이언트:

```typescript
import { google, youtube_v3 } from 'googleapis';

let youtubeClient: youtube_v3.Youtube | null = null;

export function getYoutubeClient(): youtube_v3.Youtube {
  if (!youtubeClient) {
    if (!process.env.YOUTUBE_API_KEY) {
      throw new Error('YOUTUBE_API_KEY 환경변수가 설정되지 않았습니다');
    }
    youtubeClient = google.youtube({
      version: 'v3',
      auth: process.env.YOUTUBE_API_KEY,
    });
  }
  return youtubeClient;
}
```

2. `packages/collectors/src/adapters/youtube-videos.ts`:

YouTube Data API v3의 search.list + videos.list를 조합하여 키워드 기반 영상 메타데이터를 수집한다.

구현 핵심:
- `implements Collector<YoutubeVideo>` -- Collector 인터페이스 구현
- `source = 'youtube-videos'`
- `collect()` AsyncGenerator -- 페이지 단위(maxResults=50)로 yield

API 호출 전략 (RESEARCH Pitfall 3: 쿼터 효율):
- Step 1: `search.list` -- type=video, q=keyword, publishedAfter/publishedBefore(ISO 8601), maxResults=50, order=date, regionCode=KR, relevanceLanguage=ko
  - 쿼터 비용: 100유닛/요청. maxItems 50건이면 1번 호출 = 100유닛
  - pageToken으로 페이지네이션. maxItems(D-03 기본 50건) 도달 시 중단
  - search.list는 videoId만 반환하므로 상세 정보는 videos.list로 조회

- Step 2: `videos.list` -- part=snippet,statistics, id=videoId1,videoId2,...(쉼표 구분, 최대 50개)
  - 쿼터 비용: 1유닛/요청. 50건 한번에 조회 = 1유닛
  - snippet: title, description, channelId, channelTitle, publishedAt
  - statistics: viewCount, likeCount, commentCount

YoutubeVideo 타입:
```typescript
export interface YoutubeVideo {
  sourceId: string;       // videoId
  url: string;            // https://www.youtube.com/watch?v={videoId}
  title: string;
  description: string | null;
  channelId: string;
  channelTitle: string;
  viewCount: number;
  likeCount: number;
  commentCount: number;
  publishedAt: Date | null;
  rawData: Record<string, unknown>;
}
```

쿼터 관리:
- 기본 50건 수집 시: search.list 1회(100유닛) + videos.list 1회(1유닛) = 101유닛
- 일일 10,000유닛 한도 내 약 99회 수집 가능
