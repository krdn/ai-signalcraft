# Plan 06-01: 파이프라인 진행 상태 시각화 개선

## Goal

현재 단순한 프로그레스 바 4줄로만 표시되는 파이프라인 진행 상태를:

1. **4단계 스텝 인디케이터** (수집→정규화→분석→리포트) — 현재 단계 하이라이트 + 화살표 연결
2. **소스별 수집 상세** — 네이버/유튜브/커뮤니티 각각의 수집 건수, 상태를 개별 표시
3. **12개 분석 모듈 카드 그리드** — 3×4 그리드로 각 모듈의 상태를 카드로 시각화
4. **5초 폴링 실시간 업데이트** — 진행 중일 때 자동 갱신
   으로 개선한다.

**서브 컴포넌트 분리**:

1. **`PipelineSteps`** — 4단계 스텝 인디케이터
   - 각 스텝: 아이콘 + 라벨 + 상태 텍스트
   - 스텝 간 화살표(→) 연결선
   - 현재 진행 중인 스텝: 파란색 하이라이트 + 펄스 애니메이션
   - 완료: 초록색 체크, 실패: 빨간색 X, 대기: 회색

2. **`CollectionDetails`** — 소스별 수집 상세 (접이식)
   - 수집 단계가 running 또는 completed일 때만 표시
   - 각 소스: 아이콘 + 라벨 + 수집 건수 + 상태 Badge
   - 2열 그리드 레이아웃

3. **`AnalysisModuleGrid`** — 12개 모듈 카드 그리드
   - 분석 단계가 시작되었을 때만 표시
   - 4열 그리드 (모바일: 2열, 태블릿: 3열, 데스크톱: 4열)
   - 각 카드: 상태 아이콘 + 모듈 한글명
   - 상태별 카드 색상:
     - `pending`: `bg-muted text-muted-foreground` (회색)
     - `running`: `bg-blue-50 dark:bg-blue-950 border-blue-500 animate-pulse` (파란색, 깜빡임)
     - `completed`: `bg-green-50 dark:bg-green-950 border-green-500` (초록색)
     - `failed`: `bg-red-50 dark:bg-red-950 border-red-500` (빨간색)
   - 상단에 진행률 표시: "7/12 완료" + Progress 바

4. **경과시간 표시** — `elapsedSeconds`를 "X분 Y초" 형식으로 포맷

**상태별 동작**:

- `pending`: 스텝 인디케이터만 표시 (모두 회색)
- `running` (수집 중): 수집 스텝 하이라이트 + CollectionDetails 펼침
- `running` (분석 중): 분석 스텝 하이라이트 + AnalysisModuleGrid 표시
- `completed`: 모든 스텝 초록색 + 완료 메시지 + "결과 보기" 버튼
- `failed`/`partial_failure`: 실패 스텝 빨간색 + 에러 메시지 + 재시도 버튼
